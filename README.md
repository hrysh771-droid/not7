# منصة قراءة الروايات الإلكترونية | Novel Reader Platform

منصة متكاملة لقراءة الروايات الإلكترونية باللغتين العربية والإنجليزية، تدعم الروايات الكورية والصينية واليابانية مع واجهة مستخدم حديثة وسهلة الاستخدام.

## المميزات الرئيسية

### 🌍 دعم متعدد اللغات
- واجهة باللغة العربية والإنجليزية
- دعم كامل للاتجاه من اليمين إلى اليسار (RTL)
- ترجمة ديناميكية لجميع النصوص

### 📚 إدارة الروايات
- أكثر من 150 رواية متنوعة
- تصنيفات واضحة حسب البلد (كورية، صينية، يابانية)
- تصنيفات حسب النوع (خيال، رومانسية، أكشن، فنون قتالية، إلخ)
- نظام تقييم وتعليقات

### 🔍 نظام بحث متقدم
- بحث بالعنوان، المؤلف، التصنيف، والكلمات المفتاحية
- مرشحات متعددة (البلد، الحالة، التصنيف)
- ترتيب حسب الشعبية، التقييم، التاريخ

### 📖 واجهة قراءة متطورة
- خيارات تخصيص شاملة (حجم الخط، نوع الخط، لون الخلفية)
- وضع القراءة المظلم والفاتح
- تتبع تقدم القراءة
- إشارات مرجعية

### 👤 نظام المستخدمين
- تسجيل دخول وإنشاء حساب
- ملف شخصي مع إحصائيات القراءة
- قائمة المفضلة
- تاريخ القراءة

### 🛠️ لوحة تحكم الإدارة
- إدارة الروايات والفصول
- إدارة المستخدمين
- إدارة التعليقات
- إحصائيات شاملة

## التقنيات المستخدمة

### Backend
- **Node.js** مع **Express.js**
- **TypeScript** للكتابة الآمنة
- **MongoDB** مع **Mongoose**
- **JWT** للمصادقة
- **Express Validator** للتحقق من البيانات
- **Multer** لرفع الملفات

### Frontend
- **React 18** مع **TypeScript**
- **Vite** كأداة البناء
- **Tailwind CSS** للتصميم
- **React Router** للتنقل
- **React Query** لإدارة البيانات
- **React Hook Form** للنماذج
- **Framer Motion** للرسوم المتحركة
- **React i18next** للترجمة

### الأدوات والخدمات
- **ESLint** و **Prettier** لجودة الكود
- **Concurrently** لتشغيل العمليات المتوازية
- **Nodemon** لتطوير الخادم
- **Hot Toast** للإشعارات

## التثبيت والتشغيل

### المتطلبات
- Node.js 18+ 
- MongoDB 5+
- npm أو yarn

### 1. استنساخ المشروع
```bash
git clone <repository-url>
cd novel-reader-platform
```

### 2. تثبيت التبعيات
```bash
# تثبيت تبعيات المشروع الرئيسي
npm install

# تثبيت جميع التبعيات (العميل والخادم)
npm run install:all
```

### 3. إعداد متغيرات البيئة

#### الخادم (Server)
```bash
cd server
cp .env.example .env
```

قم بتعديل ملف `.env`:
```env
NODE_ENV=development
PORT=5000
MONGODB_URI=mongodb://localhost:27017/novel-reader
JWT_SECRET=your-super-secret-jwt-key-here
JWT_EXPIRE=7d
```

#### العميل (Client)
```bash
cd client
cp .env.example .env
```

قم بتعديل ملف `.env`:
```env
VITE_API_URL=http://localhost:5000/api
```

### 4. تشغيل قاعدة البيانات
تأكد من تشغيل MongoDB على المنفذ الافتراضي (27017)

### 5. تشغيل المشروع

#### تشغيل في وضع التطوير (Development)
```bash
# من المجلد الرئيسي
npm run dev
```

هذا الأمر سيشغل:
- الخادم على `http://localhost:5000`
- العميل على `http://localhost:3000`

#### تشغيل منفصل
```bash
# تشغيل الخادم فقط
npm run server:dev

# تشغيل العميل فقط
npm run client:dev
```

### 6. بناء المشروع للإنتاج
```bash
npm run build
```

## API Endpoints

### المصادقة (Authentication)
- `POST /api/auth/register` - إنشاء حساب جديد
- `POST /api/auth/login` - تسجيل الدخول
- `POST /api/auth/logout` - تسجيل الخروج
- `GET /api/auth/me` - الحصول على بيانات المستخدم
- `PUT /api/auth/profile` - تحديث الملف الشخصي
- `PUT /api/auth/change-password` - تغيير كلمة المرور

### الروايات (Novels)
- `GET /api/novels` - الحصول على قائمة الروايات
- `GET /api/novels/:id` - الحصول على رواية محددة
- `GET /api/novels/search` - البحث في الروايات
- `GET /api/novels/popular` - الروايات الشائعة
- `GET /api/novels/featured` - الروايات المميزة
- `POST /api/novels/:id/favorite` - إضافة/إزالة من المفضلة

### الفصول (Chapters)
- `GET /api/chapters/novel/:novelId` - فصول رواية محددة
- `GET /api/chapters/:id` - الحصول على فصل محدد
- `GET /api/chapters/:id/content` - محتوى الفصل للقراءة
- `POST /api/chapters/:id/progress` - تحديث تقدم القراءة

### التعليقات (Comments)
- `GET /api/comments/novel/:novelId` - تعليقات رواية محددة
- `POST /api/comments/novel/:novelId` - إضافة تعليق جديد
- `PUT /api/comments/:id` - تحديث تعليق
- `DELETE /api/comments/:id` - حذف تعليق

### الإدارة (Admin)
- `GET /api/admin/dashboard` - إحصائيات لوحة التحكم
- `GET /api/admin/users` - إدارة المستخدمين
- `GET /api/admin/novels` - إدارة الروايات
- `GET /api/admin/comments` - إدارة التعليقات
