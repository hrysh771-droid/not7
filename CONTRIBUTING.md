# دليل المساهمة | Contributing Guide

نرحب بمساهماتكم في تطوير منصة قراءة الروايات الإلكترونية! يرجى قراءة هذا الدليل قبل البدء في المساهمة.

## كيفية المساهمة

### 1. Fork المشروع
- انقر على زر "Fork" في أعلى الصفحة
- استنسخ المشروع إلى حسابك الشخصي

### 2. إنشاء فرع جديد
```bash
git checkout -b feature/amazing-feature
# أو
git checkout -b fix/bug-fix
```

### 3. إجراء التغييرات
- قم بالتعديلات المطلوبة
- تأكد من اتباع معايير الكود
- أضف اختبارات إذا لزم الأمر

### 4. Commit التغييرات
```bash
git add .
git commit -m "Add amazing feature"
```

### 5. Push إلى الفرع
```bash
git push origin feature/amazing-feature
```

### 6. فتح Pull Request
- انقر على "New Pull Request"
- اكتب وصفاً واضحاً للتغييرات
- انتظر المراجعة والموافقة

## معايير الكود

### TypeScript
- استخدم TypeScript لجميع الملفات الجديدة
- اتبع معايير TypeScript الصارمة
- أضف أنواع البيانات المناسبة

### React
- استخدم Functional Components مع Hooks
- اتبع معايير React الحديثة
- استخدم TypeScript للـ props

### CSS/Styling
- استخدم Tailwind CSS للتصميم
- اتبع نظام الألوان المحدد
- تأكد من التصميم المتجاوب

### Backend
- استخدم Express.js مع TypeScript
- اتبع نمط MVC
- أضف validation للبيانات

## أنواع المساهمات

### 🐛 إصلاح الأخطاء (Bug Fixes)
- أصلح الأخطاء الموجودة
- أضف اختبارات لمنع تكرار الأخطاء
- اكتب وصفاً واضحاً للمشكلة والحل

### ✨ ميزات جديدة (New Features)
- أضف ميزات جديدة مفيدة
- تأكد من التوافق مع الميزات الموجودة
- أضف وثائق للميزة الجديدة

### 📚 تحسين الوثائق (Documentation)
- حسّن الوثائق الموجودة
- أضف أمثلة وأدلة الاستخدام
- ترجم الوثائق للغات أخرى

### 🎨 تحسينات التصميم (UI/UX Improvements)
- حسّن واجهة المستخدم
- أضف رسوم متحركة أو تأثيرات
- حسّن تجربة المستخدم

### ⚡ تحسينات الأداء (Performance)
- حسّن سرعة التحميل
- قلل حجم الملفات
- حسّن استهلاك الذاكرة

## إرشادات مهمة

### قبل البدء
- تحقق من Issues الموجودة
- ناقش الميزة الكبيرة قبل البدء
- تأكد من فهم متطلبات المشروع

### أثناء التطوير
- اكتب كوداً نظيفاً ومفهوماً
- أضف تعليقات للكود المعقد
- اختبر التغييرات محلياً

### بعد الانتهاء
- تأكد من عمل جميع الاختبارات
- تحديث الوثائق إذا لزم الأمر
- اكتب وصفاً واضحاً في Pull Request

## الإبلاغ عن الأخطاء

### كيفية الإبلاغ
1. تحقق من عدم وجود issue مشابه
2. استخدم القالب المحدد
3. أضف معلومات كافية لإعادة إنتاج المشكلة

### معلومات مطلوبة
- وصف المشكلة
- خطوات إعادة إنتاج المشكلة
- النتيجة المتوقعة
- النتيجة الفعلية
- لقطات شاشة إذا لزم الأمر
- معلومات النظام (OS, Browser, etc.)

## طلب ميزات جديدة

### كيفية الطلب
1. تحقق من عدم وجود طلب مشابه
2. استخدم القالب المحدد
3. اشرح الفائدة من الميزة

### معلومات مطلوبة
- وصف الميزة
- سبب الحاجة إليها
- أمثلة على الاستخدام
- اقتراحات للتطبيق

## الأسئلة الشائعة

### س: كيف أبدأ في المساهمة؟
ج: ابدأ بقراءة الوثائق، ثم اختر issue بسيط أو أصلح خطأ صغير.

### س: هل يمكنني المساهمة بالترجمة؟
ج: نعم! نحتاج ترجمات للوثائق والواجهة للغات أخرى.

### س: كيف أختبر التغييرات؟
ج: اتبع دليل SETUP.md لتشغيل المشروع محلياً.

### س: ما هي متطلبات Pull Request؟
ج: يجب أن يكون الكود نظيفاً، مع اختبارات، ووثائق محدثة.

## التواصل

### قنوات التواصل
- GitHub Issues للمناقشات التقنية
- GitHub Discussions للأسئلة العامة
- Pull Request للتعليقات على الكود

### قواعد التواصل
- كن محترفاً ومحترماً
- استخدم اللغة العربية أو الإنجليزية
- اطرح أسئلة واضحة ومحددة
- ساعد الآخرين عندما تستطيع

## شكر وتقدير

نقدر جميع المساهمات، سواء كانت:
- كود برمجي
- وثائق
- ترجمات
- تقارير أخطاء
- اقتراحات ميزات
- مراجعة كود
